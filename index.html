Overview

What is this?

The Star Distance Model uses a custom Blender add-on and 3D printing to build a physical display of a real constellation — one that reveals the true distances of its stars from Earth.

Each star is a clear acrylic rod mounted on a curved back panel. The rods point toward a virtual viewpoint so the constellation reads correctly from the front. Stand there, and you are Earth. The sphere at each rod tip sits at a position in space that corresponds to that star's real distance — closer stars reach further out toward you, distant stars sit further back.

Everything is generated from a CSV of star data. Enter your constellation, run the add-on, and it outputs a print-ready 3D model. This site walks you through building the physical model from scratch.

How It Works

The concept

01

Coordinates become positions

Every star has a right ascension (RA) and declination (Dec) — the astronomical equivalent of longitude and latitude on the sky. The add-on converts these into X/Y positions on the back panel, projecting the sky onto a curved surface so the constellation reads correctly from the front.

02

Distance becomes rod length

Each star's distance from Earth is encoded in its rod length. A star closer to Earth gets a longer rod — its sphere sits further out, closer to you the viewer. A star further away gets a shorter rod. Stand at the front and the sphere positions represent real relative distances in space.

03

Spectral class becomes colour

Stars are not all the same colour. A star's spectral class — O, B, A, F, G, K, or M — tells you its surface temperature and colour, from blue-white to deep red. Each rod base is painted with a Posca pen in the corresponding colour. The paint transmits up the clear acrylic rod to the sphere tip.

02 Preparing Your Star Data +

The add-on reads two CSV files — one for stars, one for constellation lines. You can use any constellation or any set of stars you choose.

stars.csv

One row per star. Required fields:

A few rules:

  • Southern declinations use a negative degrees value
  • Spectral class is the first letter only — drop any suffix (e.g. B2 → B)
  • Distance must be in light years

lines.csv optional

03 The Blender Add-on +

The add-on generates the full 3D model geometry from your CSV files. Install it once, point it at your data, and it outputs everything ready to print.

Installing the add-on

  1. Download constellation_model.zip from the Downloads section
  2. Open Blender and go to Edit → Preferences → Add-ons
  3. Click Install and select the downloaded zip file
  4. Enable the add-on by ticking the checkbox next to Constellation Model
  5. The add-on panel appears in the 3D viewport sidebar under N → Constellation

Generating your model

blender-ui-1.png — add-on panel (top)
blender-ui-2.png — add-on panel (bottom)

Input Files

  • Stars CSV — point this to your stars.csv file
  • Lines CSV — point this to your lines.csv file (optional)

Rod Settings

  • Min Rod Length (mm) — shortest rod in the model. Default 20mm
  • Max Rod Length (mm) — longest rod in the model. Default 500mm. If your star distance range fits within these limits, true scale is used. If not, the add-on normalises — the closest star gets 500mm, the furthest gets 20mm

Panel Settings

  • Max Panel Size (mm) — maximum width or height of the back panel. Default 250mm
  • Panel Margin (mm) — margin around the outermost stars. Default 30mm
  • Back Panel Thickness (mm) — structural thickness of the panel. Default 10mm
  • Line Width (mm) — width of recessed constellation line channels. Default 2mm

Star Labels

  • Show Star Labels — embeds star names into the panel surface
  • Show Distance — adds distance in light years below each star name
  • Label Font Size (mm) — size of embossed text. Default 10mm
  • Font — optional custom font file. Leave blank to use Blender's default

LED Box Settings

  • LED Box Depth (mm) — depth of the LED housing box. Default 45mm. If using a different M4 screw length, enter your screw length minus 5mm here (minimum 35mm)

Components

  • Generate LED Box — include the LED box in the model. Untick if building without electronics
  • Generate Preview Rods — include preview rod geometry in the viewport. For visualisation only — not printed
  • Generate Star Spheres — include the sphere tip for each rod

View

  • Switch to Layout View — rearranges all parts flat for print preparation

Click Generate Model to run. A cutting list is automatically saved to your Desktop as a text file — it includes rod lengths, spectral classes, and Posca pen colours for each star.

Exporting for print

Export each part individually as STL via File → Export → STL.

04 3D Printing +

Load your exported STL files into PrusaSlicer. The back panel, LED box, and star spheres are separate prints.

prusaslicer.png — back panel loaded in PrusaSlicer

Print recommendations

  • Printer — Prusa XL for multicolour (star labels and constellation lines in a second colour). Bambu Labs H2D for alternative materials
  • Layer height — 0.2mm
  • Infill — 7%
  • Supports — collar areas and insert holes on the back of the panel only

Back panel orientation

Place the back panel on its long side — one of the long faces flat on the print bed. The Layout View in the add-on should already orient it correctly when you export.

For multicolour prints, right-click the model in PrusaSlicer and select Split to Parts. This separates the constellation lines, star labels, and back panel body into individual objects so you can assign a different filament colour to each.

Star spheres

Split each sphere in half and print each hemisphere with the flat face down. No supports needed.

Heat set inserts

Heat set inserts fasten the LED box to the back panel (M4) and secure the acrylic rods in the collars (M3). Install M4 inserts into the back panel screw holes first, then M3 inserts into each collar.

05 Electronics optional +

Resistor calculation

Each LED requires a current-limiting resistor:

R = (V_supply − V_forward) ÷ I
R = (6.0V − 3.2V) ÷ 0.020A = 140Ω  →  nearest standard value: 150Ω

Use one 150Ω resistor per LED regardless of how many stars your model has.

Wiring

  1. Run a positive rail and a negative rail the length of the LED box
  2. At each LED position, cut the positive rail and solder in a 150Ω resistor, then the LED anode (long leg)
  3. Solder the LED cathode (short leg) to the negative rail
  4. Connect the battery box positive lead to the positive rail and negative lead to the negative rail
06 Assembly +
model-complete.jpg — completed lit model from the front

Acrylic rods

  1. Cut each rod to the length specified in the cutting list
  2. Sand the cut end starting with P120, then P240, then P400 — work toward a smooth, clear finish for best light transmission
  3. Paint the base of each rod with the Posca pen colour specified in the cutting list. The colour transmits up the rod to the sphere tip
  4. Allow the paint to dry fully before inserting

Inserting rods

  1. Insert each rod into its corresponding collar on the back panel — the collar angle is specific to each star, so refer to the cutting list for which rod goes where
  2. Tighten the M3 grub screws to secure each rod in place

Star spheres

  1. Glue the two hemisphere halves together with superglue
  2. Slide the sphere onto the rod tip — the insertion hole is offset so it engulfs the tip correctly
  3. Secure with a small drop of superglue if needed

LED box

  1. Place each LED at the end of its corresponding collar, with the LED face pointing into the rod
  2. Secure with tape
  3. Fasten the LED box to the back panel using M4 screws into the heat set inserts
  4. Connect the battery box and route the cable through the slot in the bottom wall of the LED box
  5. Switch on and check all LEDs are lit evenly

Final assembly

  1. Place the LED box against the back of the back panel, aligning the screw holes
  2. Insert M4 screws through the back of the LED box and tighten into the M4 heat set inserts in the back panel
  3. The LED box and back panel are now one assembly
